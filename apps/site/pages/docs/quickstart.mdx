# Quick Start

Get SmartChats running on your machine in under two minutes.

## Installation

<Tabs items={['macOS & Linux', 'Windows', 'Package Managers', 'Docker']}>

<Tabs.Tab>

```bash
curl -fsSL https://smartchats.ai/install | bash
```

<Callout type="info">
**Linux users.** The `tar` and `unzip` packages are required (preinstalled on most distros). If missing: `sudo apt install tar unzip` on Debian/Ubuntu, `sudo dnf install tar unzip` on Fedora. Kernel ≥ 3.10; check with `uname -r`.
</Callout>

To check that smartchats was installed successfully, open a new terminal window and run:

```bash
smartchats --version
# Output: 0.x.y
```

If you've installed smartchats but are seeing a `command not found` error, you may have to manually add the installation directory (`~/.smartchats/bin`) to your `PATH`.

</Tabs.Tab>

<Tabs.Tab>

Windows support is coming. For now, use WSL2 with the macOS/Linux install line above.

```powershell
# In a WSL2 Ubuntu shell:
curl -fsSL https://smartchats.ai/install | bash
```

Native PowerShell installer is on the roadmap — track [#W1](https://github.com/sheunaluko/smartchats/issues) for progress.

</Tabs.Tab>

<Tabs.Tab>

If you already have Node 22+ installed and want a smaller install:

```bash
npm install -g smartchats-ai
```

This ships the CLI as a Node package (~few MB), but doesn't bundle the Bun runtime or SurrealDB — `smartchats setup` will check for those and tell you what to install. For most users the `curl | sh` path above is friendlier.

Try without installing:

```bash
npx smartchats-ai setup
```

Homebrew formula and other package managers are tracked on the roadmap.

</Tabs.Tab>

<Tabs.Tab>

For containerized self-hosting:

```bash
docker run --rm -p 3000:3000 \
  -e OPENAI_API_KEY=$OPENAI_API_KEY \
  -v ~/.smartchats/data:/data \
  smartchats/smartchats-aio:latest
```

This image is built by running the standard install script inside a fresh Debian — same code path as the bare-metal install, just wrapped for environments where you want process isolation. Open [http://localhost:3000](http://localhost:3000).

<Callout type="warning">
**You must provide at least an OpenAI API key.** The container starts without one, but **the agent cannot reply** until at least one LLM provider key is configured. You'll see a loud warning in `docker logs smartchats-aio` if you forget. See [Required environment](/docs/self-host#required-environment) for the full list.
</Callout>

**Three ways to supply keys** — pick whichever fits your workflow:

| Method | When to use |
|---|---|
| `-e OPENAI_API_KEY=$OPENAI_API_KEY` flag | One-off runs, CI |
| `--env-file ~/.smartchats/.env` | Multiple vars at once, file-based |
| `-v ~/.smartchats/.env:/root/.smartchats/.env:ro` (mounted) | Same file the bare-metal CLI uses — edit, then `docker restart` to pick up changes |
| Configure via the SPA's Settings → BYO Keys page | No restart, stored in SurrealDB |

The mounted-file pattern uses the **same `.env` location convention as the bare-metal install** (`~/.smartchats/.env`), so you can switch between Docker and native and keep one config file. Create it before starting the container:

```bash
mkdir -p ~/.smartchats
cat > ~/.smartchats/.env <<EOF
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
EOF
chmod 600 ~/.smartchats/.env

docker run --rm -p 3000:3000 \
  -v ~/.smartchats/.env:/root/.smartchats/.env:ro \
  -v ~/.smartchats/data:/data \
  smartchats/smartchats-aio:latest
```

For a full self-hosting walkthrough including env-var management, persistent volumes, and non-localhost bearer auth, see [Self-Hosting](/docs/self-host).

## Upgrading

Once installed, smartchats can upgrade itself:

```bash
smartchats upgrade
```

Re-runs the installer pinned to the latest release.

**npm users:** `npm install -g smartchats-ai@latest` instead.

## Quick start

After installing, run the guided setup:

```bash
smartchats setup
```

The wizard will:

1. Check your system (bun runtime, SurrealDB binary, disk space).
2. Collect your API keys for OpenAI (required), Anthropic, Google Gemini, Serper (all optional).
3. Build whatever the stack needs.
4. Start everything and open the app on [http://localhost:3000](http://localhost:3000).

```bash
# In a different terminal — check what's running:
smartchats status

# Stop the stack:
smartchats stop

# Tail logs:
smartchats logs -f
```

That's it. Talk to the agent.
